    Planning Poker

    Projeto
    javascript

    Descrição

    Sistema para realizar estimativa de esforço de tempo, utilizando a técnica de gameficação do planning poker.

    Link de controle do projeto no Trello.

    Padrões

    Branch

    1 — type ou categoria do branch. Os types podem ser os seguintes:

    • docs: apenas mudanças de documentação;
    • feat: uma nova funcionalidade;
    • fix: a correção de um bug;
    • perf: mudança de código focada em melhorar performance;
    • refactor: mudança de código que não adiciona uma funcionalidade e também não corrigi um bug;
    • style: mudanças no código que não afetam seu significado (espaço em branco, formatação, ponto e vírgula, etc);
    • test: adicionar ou corrigir testes.

    Exemplo:

    docs/add-voting

    Commits

    Convenção de Commit
    Convenção de versionamento do sistema: Semver

    Repositório

    planning-poker.

    Casos de Usos

    Definição de APIs

    1. Criando um jogo

    POST /games

    • payload da requisição
    {
    "name": "planning poker"
}
    • exemplo de response
    {
    "id": "25351091-8e1d-4843-8aa6-1d06acd90c78",
    "name": "planning poker",
    "url": "/games/{id}"
}

    Obs.: statusCode: 201

    2. Adicionando jogadores

    POST /games/{id}/players

    {
    "name": "Lucas"
}
    • exemplo de response
    {
    "id": "25351091-8e1d-4843-8aa6-1d06acd90c78",
    "name": "planning poker",
    "players": [
        {
            "id": "2dccfa26-a027-477b-97ad-309148d5e196",
            "name": "Lucas",
            "vote": null
        }
    ],
    "url": "games/{id}"
}

    Adicionando um voto

    PATCH /games/{id}/players/{id}/vote

    *payload vote

    {
    "vote": "valueVote"
}
    • exemplo de response
    {
    "id": "25351091-8e1d-4843-8aa6-1d06acd90c78",
    "name": "planning poker",
    "players": [
        {
            "id": "2dccfa26-a027-477b-97ad-309148d5e196",
            "name": "Lucas",
            "vote": "3"
        }
    ],
    "url": "games/{id}"
}

    Buscando os votos

    GET /games/{id}

    • payload revelar os votos

    • exemplo de response

    {
    "id": "25351091-8e1d-4843-8aa6-1d06acd90c78",
    "name": "planning-poker",
    "players": [
        {
            "id": "2dccfa26-a027-477b-97ad-309148d5e196",
            "name": "Lucas",
            "vote": "3"
        },
        {
            "id": "2dccfa26-a027-477b-97ad-309148d5e196",
            "name": "Renato",
            "vote": "5"
        }
    ],
    "url": "games/{id}"
}

      Gerar média de votos?
